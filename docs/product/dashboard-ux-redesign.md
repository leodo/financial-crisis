# 风险面板 UX 改造设计

状态：`Draft`

最后更新：2026-05-30

## 1. 问题

当前面板把风险分数直接展示为一个大数字，用户容易产生三个误解：

- 以为 `98.5` 就等于“已经发生金融危机”。
- 不知道 `0-100` 是风险强度分，不是概率。
- 不知道单指标 `100` 是历史样本分位极端，不是指标本身满分。

同时，demo 数据过短，很多指标最新值正好是样本最高或最低，导致风险分大量为 `100`，不适合演示。

## 2. 改造目标

面板需要从“显示分数”改成“解释风险判断”：

1. 先告诉用户当前结论。
2. 再解释分数含义。
3. 再提供历史危机对照。
4. 最后让用户下钻到指标和数据质量。

## 3. 总体信息架构

```text
总览页
  当前结论
    当前等级
    当前分数
    是否进入危机判定
    主要原因
  分数标尺
    L0 0-29 正常
    L1 30-49 观察
    L2 50-69 压力
    L3 70-84 预警
    L4 85-100 危机态
  历史对照
    当前分数
    2008 全球金融危机峰值
    2020 流动性冲击峰值
    2023 区域银行危机峰值
  维度和指标
    结构性风险
    触发性风险
    维度热力
    指标表
```

## 3.1 第一版落地范围

- 总览页顶部增加解释条，先消除“风险分=危机概率”的误读。
- 总览页首屏拆成三个区域：当前结论、分数标尺、历史危机对照。
- 历史危机对照从回测场景接口读取，默认包含 `2007-2009`、`2020`、`2023` 三类压力样例。
- 指标表增加历史分位和方向规则解释，让用户知道单指标分数来自自身历史区间，而不是跨指标绝对值比较。
- Demo 数据加入历史极端样本，避免最新值总是样本极值导致大面积 `100` 分。

## 4. 文案原则

- 明确说明：分数是风险强度，不是金融危机概率。
- 明确说明：L4 表示系统处于危机态或接近历史危机区间，不等于官方宣布危机。
- 指标表要解释分数来源，例如“处于历史高分位”“低于历史多数样本”“双向偏离”。
- 对 demo 数据要标记为“校准样例”，避免用户误认为实时生产数据。

## 5. Demo 数据调整

Demo 数据应包含历史极端场景作为参照：

- 2008 信用利差和金融压力高点。
- 2020 VIX 和流动性冲击高点。
- 2023 银行压力场景。

当前样例应落在 `L2 Stress` 到 `L3 Warning` 附近，避免默认就是 `L4 Crisis`。

## 6. 触发解释

等级解释：

- `L0`：正常，未见系统性压力。
- `L1`：观察，少数指标开始偏离。
- `L2`：压力，多个指标提示风险积累。
- `L3`：预警，多个维度共振，需要人工关注。
- `L4`：危机态，分数进入历史危机附近，需要结合事件确认。

第一版不做“金融危机已发生”的自动结论。真正的危机判定需要：

- 总分进入 L4。
- 至少两个核心维度 L3 以上。
- 市场/流动性/事件中至少一个触发类维度 L3 以上。
- 数据质量不低于 C。
- 人工或事件层确认。
