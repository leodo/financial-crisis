# 方法页设计

状态：`Draft`

最后更新：2026-05-30

## 1. 目标

设计一个面向用户和研究者的方法页，解释概率、风险强度、posture 和可信度分别代表什么。

## 2. 页面目的

- 降低用户误读
- 展示方法透明度
- 解释为什么系统不是黑盒

## 3. 页面结构

### 3.1 当前方法版本

- score method
- feature set
- label version
- model version
- calibration version

### 3.2 输出解释

- 风险强度是什么
- 概率是什么
- posture 是什么
- conviction 是什么

### 3.3 历史可靠性

- 命中率
- 提前量
- 校准曲线

### 3.4 数据覆盖与限制

- 免费数据范围
- 当前缺失项
- 数据修订风险

## 4. UI 原则

- 用图表解释，不堆大量公式
- 重点回答“这个系统为什么值得信”
- 明确局限，不夸大能力
