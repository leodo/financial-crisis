# 系统可行性分析

状态：`Draft`

最后更新：2026-05-31

## 1. 目标

回答三个问题：

1. 这个“美国主线金融危机概率评估系统”是否能仅靠免费数据落地。
2. 如果能落地，最可信、最可执行的形态是什么。
3. `1990`、`1987` 这类历史跨度问题，应该怎样进入系统而不是靠主观猜。

## 2. 先说结论

可以落地，但要把目标收紧到“可解释、可审计、可回测的危机风险评估系统”，而不是承诺一个能稳定提前一周精确预测所有危机的黑盒模型。

最可行的路线不是：

- 一上来就做大而全监督学习；
- 直接把免费数据训练成一个高置信度自动交易信号；
- 看到高概率就机械清仓。

最可行的路线是：

1. 用免费官方数据先做稳定的数据底座和历史回放。
2. 用规则风险层给出结构脆弱性、触发压力、外部冲击三层解释。
3. 在此基础上叠加一个小而可解释的概率层，只做 `5d / 20d / 60d` 前瞻风险。
4. 再叠加历史类比、置信度、动作手册，服务“减仓 / 对冲 / 防守 / 再入场”决策。

## 3. 可行与不可行的边界

### 3.1 当前可行

- 美国宏观、利率、期限利差、信用代理、金融压力、波动率、美元兑日元等核心时序。
- 银行业公告、监管事件、风险关键词的事件聚合。
- 基于历史危机窗口的滚动审计、误报分析和 posture 回放。
- “当前离风险有多远”这种分层判断：
  - `months`
  - `weeks`
  - `days`
  - `now`

### 3.2 当前不可行或不该承诺

- 分钟级或逐笔级全市场实时危机监控。
- 用免费源稳定获得完整 CDS、机构资金流、银行内部流动性。
- 只靠监督学习就给出可信的“危机发生概率”并直接驱动自动清仓。
- 把所有风险事件都压成一个单标签，然后指望模型自己学明白。

## 4. 免费数据到底能追多早

结论先写清楚：

- `1990` 不是历史数据的物理上限。
- `1990` 是第一阶段“统一日频多因子面板”的稳妥基线。
- `1987` 不是没数据，而是数据维度没有 `1990` 以后那么齐全。

### 4.1 目前已经确认可免费拿到的代表性历史跨度

| 数据 | 免费官方入口 | 已确认历史跨度 | 说明 |
|---|---|---|---|
| `FEDFUNDS` | FRED Graph CSV | `1954-07` 起 | 联储政策利率长历史可用 |
| `BAA10Y` | FRED Graph CSV | `1986-01-02` 起 | 可作为信用压力长历史代理 |
| `VIXCLS` | FRED Graph CSV | `1990-01-02` 起 | 这也是很多快变量从 `1990` 起步的重要原因 |
| `DEXJPUS` | FRED Graph CSV | `1971-01-04` 起 | 日元汇率长历史可用，可覆盖 `1987` |
| U.S. Treasury Daily Curve | Treasury XML | `1990-01-02` 起 | 日频收益率曲线统一起点 |
| World Bank 年频宏观 | World Bank API | 常见为 `1960` 起 | 适合慢变量背景，不适合日频触发 |
| SEC EDGAR metadata | SEC APIs | 实务上可追到 `1994` 以后 | 对 `1987` 无法提供事件层支持 |

### 4.2 为什么当前设计大量使用 `1990`

因为从系统工程角度看，`1990` 左右开始才比较容易拿到一组连续、免费、日频、还能互相对齐的关键快变量：

- `VIX`
- 日频 Treasury curve
- 更稳定的市场类免费时序
- 后续还能逐步接上现代事件层和公告层

所以：

- `1990` 适合做第一版正式日频回测基线。
- `1987` 可以纳入扩展场景，但通常需要代理特征包，而不是直接套用 `1990+` 的完整特征集。

## 5. 1987 大股灾到底算不算“危机”

这不是单纯历史常识问题，而是标签设计问题。

本系统更适合把风险事件拆成三类：

1. `systemic_credit_banking_crisis`
2. `acute_market_liquidity_crash`
3. `rate_shock_or_policy_dislocation`

按这个分类：

- `2008` 更接近系统性信用/银行危机。
- `2020-03` 更接近急性流动性和全市场冲击。
- `2023` 更接近区域银行和存款压力。
- `1987-10` 更适合归到 `acute_market_liquidity_crash`，也就是“市场崩盘/流动性冲击场景”。

因此，`1987` 应该纳入系统，但它不应该和 `2008` 完全视作同一种标签。

推荐做法：

- 在 `historical analog` 中强制纳入 `1987`；
- 在 `p_5d / p_20d` 的急性冲击标签中纳入 `1987`；
- 在“系统性银行危机主标签”里，不直接把 `1987` 和 `2008` 混成同一种正例。

## 6. 训练这件事到底靠不靠谱

### 6.1 主要问题不在“数据行数少”，而在“有效正例少”

就算把日频数据从 `1990` 拉到今天，样本天数很多，但真正关键的正例窗口仍然很少。

美国主线真正值得重点建模的高压场景，大致只有这些：

- `1987`
- `1998 LTCM / 俄罗斯违约外溢`
- `2000-2002` 资产泡沫破裂后的信用与股市压力
- `2007-2009`
- `2011` 美欧融资压力
- `2020-03`
- `2022` 快速加息与久期冲击
- `2023` 区域银行压力

这意味着：

- 高复杂度模型很容易过拟合；
- 任意一个标签定义改动，结果都可能大变；
- 不能把单次回测命中当成模型已可靠。

### 6.2 免费数据训练的真正限制

| 限制 | 对模型的影响 |
|---|---|
| FRED Graph CSV 默认没有 vintage | 严格 point-in-time 重建受限，容易引入轻微未来函数 |
| 事件发布时间与交易日对齐不完全 | `p_5d` 的短窗标签噪声更大 |
| 新闻源噪声高 | 不能让新闻信号单独决定 posture |
| 某些高价值信用数据只能拿代理 | 模型解释力和稳定性受限 |
| 不同年代可用特征不一致 | `1987` 与 `2023` 难以直接同维度建模 |

### 6.3 因此什么样的模型才可信

可信的不是“大模型”或“复杂模型”，而是：

- 小特征集；
- 强正则；
- 清晰标签；
- 时间切分；
- 可校准；
- 能解释为什么升风险；
- 知道自己什么时候证据不足。

第一阶段建议坚持：

- 逻辑回归 / Probit / 简化 hazard 风格模型；
- `Platt` 或 `isotonic` 校准；
- 数据覆盖率和样本相似度驱动的 `conviction` 降权；
- 规则层保留 override 和降级逻辑。

## 7. 当前代码处在什么位置

当前仓库已经打通了一条可运行的 formal serving 链路，但它还不是最终形态。

当前 formal bundle 的现实定位是：

- 用已落库 `prediction snapshots` 训练；
- 用美国危机起点目录打标签；
- 先验证 `train -> publish -> activate -> serve -> audit -> rollback` 整条工程链路。

这条链路是有价值的，但它还是“过渡版”：

- 还不是最终的 raw point-in-time feature store 正式模型；
- 还没有完整 vintage 审计；
- 当前代码中的 crisis start 仍只覆盖 `2007-08-01`、`2020-02-24`、`2023-03-08`；
- `1987`、`1998`、`2011` 这类扩展场景还没有真正纳入正式标签流水线。

所以现在更准确的说法应该是：

“系统已经具备概率评估工程骨架，但最终的正式概率模型仍需建立在更完整的数据可见性和场景标签之上。”

## 8. 最可行的产品路线

### 8.1 第一阶段：`1990+` 统一日频基线

目标：

- 先把 `1990` 至今的美国核心免费日频面板做稳。
- 保证 `VIX / 利率曲线 / 信用代理 / 金融压力 / USDJPY / 事件聚合` 可持续回填。
- 做滚动审计、误报审计、posture 回放。

产出：

- 规则风险层
- 过渡概率层
- 历史类比
- 决策动作面板

### 8.2 第二阶段：`1987 / 1998 / 2011` 扩展场景包

目标：

- 为早期历史场景建立“代理特征集”。
- 明确哪些特征在早期年代缺失，哪些可以降级替代。
- 把 `1987` 作为急性市场崩盘样本，而不是强行并入银行危机主标签。

产出：

- 场景目录扩展
- 早期历史特征覆盖矩阵
- 扩展版 historical analog

### 8.3 第三阶段：正式 raw feature + label pipeline

目标：

- 从原始观测值生成 point-in-time feature store；
- 严格区分 `published_at / observed_at / ingested_at`；
- 重新训练 formal bundle，而不是继续依赖 `prediction snapshots` 过渡训练。

### 8.4 第四阶段：把模型真正变成决策支持

目标：

- 把 posture 和持仓动作手册连接到真实组合规则；
- 做“减仓/留底仓/买保护性认沽”的历史策略回放；
- 评估误杀成本和漏报成本，而不是只看 AUC。

## 9. Go / No-Go 判断

满足以下条件，才适合把 formal 概率层往“正式决策输入”推进：

1. `1990+` 核心日频面板覆盖率稳定。
2. 至少完成一版 point-in-time 特征快照。
3. 标签目录不只包含 `2008 / 2020 / 2023` 三个主场景。
4. 全历史滚动审计能稳定区分：
   - 危机前命中
   - 危机中命中
   - 受保护压力窗口
   - 纯误报
5. 概率高低能明显改善 posture 决策，而不是只是把现有规则分重命名。

如果以上条件做不到，就不应该把它包装成“可以直接决定清仓”的正式模型。

## 10. 当前最合理的研发判断

这个项目不是不可行。

真正可行的版本是：

- 美国主线；
- 免费官方数据优先；
- 日频为主；
- 规则风险层打底；
- 小而可解释的概率层补充；
- 1987 等早期事件作为扩展场景而不是强行做统一全特征训练。

换句话说，应该继续做，但路线要务实：

先把“可信评估系统”做扎实，再把“正式概率模型”做严格。
