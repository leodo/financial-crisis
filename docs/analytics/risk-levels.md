# 风险等级设计

状态：`Draft`

最后更新：2026-05-30

## 1. 目标

定义系统输出的风险等级、触发条件、升级降级规则和前端展示语义。风险等级是用户理解系统判断的核心，不应只展示一个数字分数。

## 2. 等级定义

| 等级 | 名称 | 分数区间 | 含义 |
|---|---|---:|---|
| L0 | `Normal` 正常 | 0-29 | 指标处于历史正常区间 |
| L1 | `Watch` 观察 | 30-49 | 部分指标走弱，需要关注 |
| L2 | `Stress` 压力 | 50-69 | 多个维度出现压力，风险积累 |
| L3 | `Warning` 预警 | 70-84 | 系统性风险明显升高，需要行动 |
| L4 | `Crisis` 危机 | 85-100 | 已处于危机或高度危机状态 |

## 3. 等级语义

### L0 Normal

含义：

- 当前风险处于低位。
- 没有主要维度处于异常压力。

展示：

- 绿色。
- 不生成预警事件。

### L1 Watch

含义：

- 单个维度或少数指标开始恶化。
- 尚不足以判断系统性压力。

展示：

- 蓝色或浅黄色。
- 可以生成观察记录，但不触发高优先级通知。

### L2 Stress

含义：

- 市场、信用、流动性或宏观中至少一个维度达到明显压力。
- 需要分析主要贡献指标。

展示：

- 黄色。
- 生成压力事件。

### L3 Warning

含义：

- 多维度共振，或者单个核心维度极端恶化。
- 系统进入高风险状态。

展示：

- 橙色。
- 生成预警事件，要求人工确认。

### L4 Crisis

含义：

- 已出现严重市场压力、融资压力或事件冲击。
- 危机状态可能已经发生或非常接近。

展示：

- 红色。
- 生成高优先级事件，保留审计和解释。

## 4. 触发规则

默认规则：

```text
score < 30       -> L0
30 <= score < 50 -> L1
50 <= score < 70 -> L2
70 <= score < 85 -> L3
score >= 85      -> L4
```

增强规则：

- 如果数据质量等级低于 C，整体等级最高只能显示为 `Data Limited` 包装状态，除非已有高质量核心指标触发 L3/L4。
- 如果 L3/L4 仅由单个低质量指标推动，不自动触发正式预警。
- 如果触发维度少于最低要求，等级显示为候选等级，等待确认。

## 5. 多维度确认

L3 默认需要满足至少一个条件：

- 整体分数达到 L3，且至少两个核心维度达到 L2。
- 任一核心维度达到 L4，且至少一个触发类维度达到 L2。
- 事件类指标出现重大事件，且市场压力达到 L2。

L4 默认需要满足至少一个条件：

- 整体分数达到 L4，且至少两个核心维度达到 L3。
- 市场压力或流动性维度达到极端阈值，且事件类指标确认重大冲击。
- 人工确认外部已发生危机事件，并将系统状态切换为 Crisis。

## 6. 升级和降级

升级：

- L0 到 L1：单周期即可。
- L1 到 L2：连续 2 个周期或单周期超过 60。
- L2 到 L3：连续 2 个周期，或单周期超过 78 且多个维度共振。
- L3 到 L4：单周期超过 90，或重大事件触发。

降级：

- 需要连续 3 个周期低于当前等级下沿。
- L3/L4 解除需要保留事件记录，并进入观察状态。

## 7. 等级与通知

| 等级 | 是否通知 | 是否需要确认 | 是否进入预警记录 |
|---|---|---|---|
| L0 | 否 | 否 | 否 |
| L1 | 可选 | 否 | 可选 |
| L2 | 是，低优先级 | 否 | 是 |
| L3 | 是，中高优先级 | 是 | 是 |
| L4 | 是，高优先级 | 是 | 是 |

## 8. 前端展示要求

风险等级展示必须包含：

- 当前等级。
- 当前分数。
- 最近一次变化。
- 等级触发原因。
- 主要贡献指标。
- 数据质量提示。
- 与历史危机时期的相似度或分位参考。

禁止只显示“危机概率”而不显示解释。
