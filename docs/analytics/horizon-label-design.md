# 危机窗口与标签设计

状态：`Draft`

最后更新：2026-05-31

## 1. 目标

定义“危机概率”背后的标签体系，确保系统输出的 `5d/20d/60d` 概率有明确含义，而不是把解释性风险分硬包装成概率。

## 2. 设计原则

- 标签必须独立于模型输出。
- 标签优先基于可验证事件和市场结果。
- 同时支持急性冲击和中期风险积累。
- 标签粒度要能支撑 `5d/20d/60d` 三个 horizon。

## 3. 事件家族

### 3.1 急性流动性冲击

用于 `p_5d` 和部分 `p_20d`。

典型表现：

- VIX 急升
- 美股快速深跌
- 信用利差急剧走阔
- 融资条件显著恶化

### 3.2 银行/融资压力

用于 `p_20d` 和 `p_60d`。

典型表现：

- 银行业公告和 8-K 密集出现
- 存款流失或融资成本异常
- 监管、救助、折价融资等事件

### 3.3 广义风险资产危机窗口

用于 `p_20d` 和 `p_60d`。

典型表现：

- 股票、信用、利率、波动率多资产共振
- 风险资产进入深回撤
- 市场从压力阶段升级到危机阶段

## 4. 标签层级

系统保留三层标签。

### 4.1 场景标签

由研究和回测使用：

```text
scenario_id
name
event_family
crisis_start
crisis_peak
crisis_end
pre_warning_start
pre_warning_end
source_note
```

### 4.2 前瞻危机标签

由模型直接使用：

```text
label_5d
label_20d
label_60d
```

定义为：

```text
label_H = 1
当且仅当从评估日 t 往后 H 个交易日内，首次进入定义好的 crisis_start 或 acute_start
```

### 4.3 动作窗口标签

用于研究“是否已经进入可以开始减仓 / 对冲 / 防守的窗口”：

```text
action_label_5d
action_label_20d
action_label_60d
```

第一版 bounded action window 规则：

- `5d`：围绕 `acute_start` 或 `crisis_start`，窗口约为 `[-10d, +7d]`
- `20d`：围绕 `crisis_start`，窗口约为 `[-35d, +20d]`
- `60d`：围绕 `crisis_start`，窗口约为 `[-90d, +30d]`

补充规则：

- 若场景与该 horizon 不完全对齐，也保留 fallback 动作窗口，避免时间切分后整段 split 没有正样本；
- 但训练时会通过 `scenario role` 权重把这些 fallback 正样本降权；
- `action_label_*` 不等于“危机已经发生”，而是“动作级预警窗口已经打开或刚被验证”。 

## 5. 时间窗口定义

### 5.1 5 日窗口

目标：

- 捕捉“很快可能要出事”的状态
- 更偏向流动性和市场触发

正例：

- 未来 5 个交易日内进入急性冲击窗口
- 未来 5 个交易日内发生银行或流动性事件并触发市场共振

### 5.2 20 日窗口

目标：

- 捕捉“未来几周可能显著恶化”的状态

正例：

- 未来 20 个交易日内进入危机窗口
- 未来 20 个交易日内至少出现一次持续数日的系统性高压阶段

### 5.3 60 日窗口

目标：

- 捕捉“未来数月存在较高进入危机区间概率”的状态

正例：

- 未来 60 个交易日内从脆弱性积累进入明确压力/危机阶段

## 6. 危机窗口的客观定义

每类事件都要尽量依赖可复核条件。

### 6.1 市场型危机窗口

候选判定规则：

- 标普 500 在 20 个交易日内回撤超过阈值
- VIX 超过历史高分位阈值
- 高收益债 OAS 显著走阔
- 至少两个触发条件同时满足

### 6.2 银行型危机窗口

候选判定规则：

- 银行业高频事件显著增加
- 资金和流动性指标进入极端区间
- 伴随明显市场压力

### 6.3 混合型危机窗口

候选判定规则：

- 市场、融资、事件三者中至少两类共振

## 7. 样本生成流程

```mermaid
flowchart LR
    A["历史原始数据"] --> B["场景表"]
    B --> C["定义 crisis_start / acute_start"]
    C --> D["按交易日展开样本"]
    D --> E["生成 5d/20d/60d 标签"]
    E --> F["回测与训练"]
```

## 8. 样本切分

- 训练样本必须按时间切分，不允许随机打散。
- 方法调参和最终评估分离。
- `2008`、`2020`、`2023` 至少要有独立评估段。

## 9. 评估指标

除了 AUC，更看重：

- 提前量
- Precision@high-risk
- 高概率区间的命中率
- 误报持续天数
- 校准误差

## 10. 第一批场景

| 场景 | 危机开始日期 | 用途 |
|---|---|---|
| `us_gfc_2008` | 2007-08-01 | 信用与银行压力 |
| `us_covid_liquidity_2020` | 2020-02-24 | 急性流动性冲击 |
| `us_rate_shock_2022` | 2022-01-03 | 利率与资产重估 |
| `us_regional_banks_2023` | 2023-03-08 | 银行和存款风险 |

第一批场景是“先把主线跑通”的最小集合，不代表完整历史目录。

### 10.1 第二批扩展场景

| 场景 | 建议开始日期 | 建议标签家族 | 作用 |
|---|---|---|---|
| `us_black_monday_1987` | 1987-10-14 | `acute_market_liquidity_crash` | 急性市场崩盘与流动性冲击类比 |
| `us_ltcm_1998` | 1998-08-17 | `acute_market_liquidity_crash` | 杠杆去化与外溢冲击 |
| `us_funding_stress_2011` | 2011-07-29 | `mixed_systemic_stress` | 主权、融资与风险偏好共振 |

设计原则：

- `1987` 应纳入系统，但不应与 `2008` 视为同一种银行信用危机标签。
- 第二批扩展场景优先服务：
  - `historical analog`
  - `p_5d / p_20d` 急性冲击标签
  - 受保护压力窗口目录

## 11. JPY Carry 的标签作用

日元套息模块第一阶段不单独定义独立危机标签。

它的用途是：

- 作为 `external_shock` 特征增强美国主线模型
- 在高脆弱性阶段提高 `p_5d` 和 `p_20d`
- 作为历史相似度中的辅助解释项

## 13. 后续开发提示

- 先手工维护第一版场景表。
- 再把场景规则程序化。
- 样本生成必须支持 point-in-time 数据视角。
- 标签文档一旦调整，必须提升版本号并重跑回测。
- 当前已经验证：单独把 formal 主模型整体切到 `action_label_*` 仍不足以通过 runtime guard。
- 下一步应进入 `separate actionability head / episode objective / dual-head fusion`，而不是继续只改同一套单头概率标签。
